面试官：你有什么想问的？

发表于 2020-11-09 更新于 2021-10-11 分类于 面试题 Disqus：

技术

  • 任意页面布局和代码逻辑是否可以不运行App主逻辑进行直接运行到真机调试看效果？
  • 在壳工程可以调试已经发布为aar依赖的子工程的源码吗？
  • 主工程和多个子工程依赖相同的库时，如何同时在不同工程升级这个库？
  • 多个人在同一个子工程发布aar怎么保证发布时是最新代码？并且版本号没有冲突？
  • 公共基础库有新的改动，API不兼容之前的版本了，这个库的新的版本号是怎么定的？
  • 子工程网络请求操作是怎么调试验证的？
  • UI数据绑定实体类、网络请求序列化实体类、数据库ORM实体类是同一个类吗？
  • Activity、Fragment里会有数据处理逻辑吗？还是纯粹的视图和数据绑定？
  • 获取当前系统时间是怎么封装调用的？
  • 怎么统一限制App总线程数的？
  • 怎么统一禁止主线程做数据库操作的？
  • 依赖注入框架用的是什么？
  • 单元测试覆盖率有强制要求吗？
  • 单元测试框架用的是哪些？
  • 只运行指定的几个功能模块的单元测试集是怎么做的？
  • UI是怎么做测试的？
  • 黑盒测试用的什么框架？
  • 每引入新的依赖库，会要求对性能和体积进行评估记录吗？
  • 客户端会经常做A/B test吗？测试的都是什么内容？
  • APK体积和用户新增数量有量化指标监控吗？
  • 编译完整的项目的时间会超过5分钟吗？
  • 有搭建自己的maven服务器缓存mavenCenter里的依赖库吗？
  • git push代码后，CI会做编译校验和增量lint检查吗？
  • git push代码后，会有工具自动检测编译时间和Apk体积的变化吗？
  • 对App有权限申请的监控吗？可以追查到权限申请的代码位置吗？
  • 统计、日志等基础SDK存储在外存的数据有压缩和加密吗？什么时候触发写文件的？
  • 开发人员在代码里手动添加打点统计的点之后是怎么自测的？
  • 怎么保证每次发版时手动统计埋点都正确？
  • 废弃的手动统计埋点是怎么清理的？
  • 怎么保持Android和iOS埋点规则一致的？比如页面访问时长的统计规则是怎么定的？
  • Fragment的页面访问事件是无埋点全自动统计吗？
  • RecyclerView的item曝光事件是怎么埋点的？怎么传自定义参数的？
  • 怎么避免重复埋点的？
  • 怎么避免埋点丢失的？
  • 批量上传埋点数据，总是返回400，怎么处理数据的？
  • 怎么避免打印无效日志的？调试看信息的日志在功能开发完成后会删除吗？
  • 产线包里输出的每一个error或warning的日志是否能直接定位到问题？是否都有自定义的错误码？
  • 日志内容对于测试、客服等不懂开发的人是否可读？
  • 怎么避免把隐私信息写到日志和统计SDK里去的？
  • 项目的多渠道打包是怎么实现的？
  • 项目有在用热修复吗？用的是哪个方案？
  • ConstraintLayout用的人多吗？
  • 自定义View会考虑从右到左的布局吗？
  • 多语言下字符串长度过长，会针对不同语言设计多个布局吗？
  • 给TextView设置纯数字内容，是调用数字类的toString()方法转字符串吗？
  • 设计图中UI控件的样式在代码中有主题样式逐一对应吗？
  • 每个类名有@author的注释吗，author名称是邮箱地址吗？
  • App发生崩溃或卡顿，会立刻自动建一个Jira任务并附上相关错误信息指派到个人吗？
  • Jetpack Paging、Compose在用吗？
  • Kotlin协程在用吗？启动协程都是通过GlobalScope调用的吗？callbackFlow在用吗？
  • git要求原子化提交吗？
  • git message要求写出每一个代码改动的原因吗？
  • CI对git message有强制的格式检查约束吗？
  • 同一个commit要在多个分支上手动cherry-pick好几次吗？

团队

  • 组里有多少人有独立开发的App作品？日活有多少？累计下载量有多少？
  • 组里有多少人做过开源项目？做的是哪方面的内容？
  • 组里有多少人会写单元测试？多少人了解函数式编程？
  • 组里有多少人同时掌握Android和iOS？
  • 组里有多少人同时掌握5门以上编程语言？
  • 部门领导或技术总监会定期参与一线开发的Code Review吗？
  • 您认为在组内推广单元测试的困难在哪？
  • 您认为在组内推广Kotlin的困难在哪？
  • 您认为用Groovy或Scala做Android开发会有什么问题？
  • 您认为Kotlin、Groovy、JavaScript主要的区别是什么？
  • 产品经理会对功能需求描述给出详细的测试用例文档吗？有工具把产品文档生成测试代码吗？
  • DevOps团队里的人能熟练掌握Gradle和Groovy吗？
  • 每个人可以看到所有仓库的代码吗？
  • 您负责的Git仓库会有别人发现问题过后给你提交补丁吗？
  • 申请一个新的Git仓库有什么要求？
  • 各自负责的Git仓库里的分支管理是自由发挥吗？还是要遵循统一标准？
  • 公司内部技术分享文档的评论数一般会超过5条吗？
  • 对于一线员工的意见反馈，怎么收集和解决落实的？
  • 加班多吗？加班的原因是什么？
  • 组里一到两年内职级晋升的人数比例是多少？
  • 组里一到两年内涨薪的人数比例是多少？最低和最高薪资涨幅是多少？

办公环境

  • 办公椅有靠枕吗？
  • 工位之间有隔板吗？
  • 工位附近有白板吗？
  • 业务相关的上下游部门员工是否坐在同一个区域？
  • 每一层办公区是一个大开间吗？从座位上站起来是否可以看到任何一个人？
  • 每一层只有一个卫生间吗？
  • 每一层办公区只有一两条长走道吗？
  • 部门领导或技术总监有单独的办公室吗？
